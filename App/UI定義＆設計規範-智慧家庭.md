# 智慧家庭 - UI定義＆設計規範

## 整體UI採兩層架構：
- 歡迎頁面：
  首頁為歡迎頁面，列示所有頁面的集合，該按鍵以設備名稱或是空間名稱為主（以下將以設備按鍵稱之），能夠表示點選該按鍵要進入的控制功能項目的『功能頁面』。

 ![N|Solid](https://md.smart-ehome.com/amma/manual/img/uidesign/1.png)

- 功能頁面：
  功能頁面由兩個視窗框架結合，為功能按鍵集合視窗及設備按鍵集合視窗。
  功能按鍵排列依照該設備功能項目Layout定義排列，若該設備未定義，則依實際功能鍵數量對照每頁按鍵數量設計排列。
 
  ![N|Solid](https://md.smart-ehome.com/amma/manual/img/uidesign/4.png)

### Step By Step
透過step by step 預設將以該設備所有功能為一個控制頁面，因此可以很快速產生使用者控制介面。

### UI設計要點
一款UI設計圖與組態包含如下要點：

#### ICON
* 製作所有定義『設備清單』iCON圖示（128x128),檔名為3位數iconID命名。
```
例 ：
      乾衣機 06D.png
```

#### 頁面設計

##### 歡迎頁面設計(000_bg.png,000_bg_up.png,000_bg_dn.png)
> 設計該頁排滿設備按鍵的排列，按鍵設計不需要有文字與圖，系統會以該設備的icon填入，上下按鍵排列需考量好流設備名稱的文字。
當實際運用時，該頁設備排滿，往左滑動可切換至第二頁。

  ![N|Solid](https://md.smart-ehome.com/amma/manual/img/uidesign/2.png)

  ![N|Solid](https://md.smart-ehome.com/amma/manual/img/uidesign/3.png)

##### 功能頁面設計(xxx_bg.png,xxx_bg_up.png,xxx_bg_dn.png)
> 頁面除底圖與該設備功能按鍵外，必須含有兩項目：
 - HOME 按鍵：用來返回首頁（歡迎頁面）
 - Devices Bar : 該矩形長方將含有所有設備按鍵，用來切換至該設備項目頁的功能。

 ![N|Solid](https://md.smart-ehome.com/amma/manual/img/uidesign/5.png)

* 製作所有定義『設備功能清單』頁面（底圖＋該頁按鍵UP+該頁按鍵Down）含排列Layout檔。
```
  例： 以『乾衣機』為例
      該頁面底圖 06D_bg.png
      該頁面所有功能按鍵排列圖,按鍵平常呈現圖樣,底圖採透明色 06D_bg_up.png
      該頁面所有功能按鍵排列圖,按鍵按下呈現圖樣,底圖採透明色 06D_bg_dn.png
      該頁面所有功能按鍵Layout檔 06D.json
```

* 製作標準按鍵排列Layout檔(default)。
  (每頁分別設計可以放置以下數量按鍵：1,2,3,4,5,6,8,10)
```
  例： 
      預設按鍵排列（必要,共八種排列）
      該頁面只有一個功能按鍵Layout檔 000_1.json
      該頁面有二個功能按鍵Layout檔 000_2.json

      以『燈光』為例
      該頁面只有一個功能按鍵Layout檔 011_1.json
      該頁面有二個功能按鍵Layout檔 011_2.json
      該頁面有五個功能按鍵Layout檔 011_5.json
      該頁面有八個功能按鍵Layout檔 011_8.json
      該頁面有十個功能按鍵Layout檔 011_10.json
```

## 附錄（一） 設備項目定義
* APP呈現設備圖示依照 icon ID 顯示。
* 所有制定的設備,須製作ICON圖示 128 x 128 （Pixels）
* 該設備功能未明確定義將依據每頁定義按鍵數量排列。
( 設計8頁按鍵排列Layout,每頁分別可以放置以下數量按鍵：1,2,3,4,5,6,8,10 )

## 附錄（二） 功能項目定義
* APP呈現功能按鍵上的圖示依照 icon ID 顯示。
* 所有制定設備功能,須製作相對應ICON圖示 （128x128）
* 各設備功能的排列Layout該功能鍵須符合該功能ID。


#### 編碼格式(2位數16進制)
|②|①|
|:---:|:---:|
|功能ID|功能ID|

＊icon ID = ②① (2位數)
